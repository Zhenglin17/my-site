# 一、项目目标（非常重要）

这是一个**个人网站项目（Personal Website）**，目标分为两条主线：

## 1️⃣ 求职导向（信息高效传达）

面向：
- HR
- 招聘经理
- 快速浏览者

特点：
- 简洁、直接、信息密度高
- 类似 GitHub Pages 风格
- 快速展示：
  - 项目经历
  - 技术能力
  - 关键信息

👉 目标：**让别人 30 秒内了解我是谁、做过什么**

---

## 2️⃣ 个人传记 / Full Story 页面（`/story`）

面向：
- 愿意深入了解我的人
- 技术/非技术观众

### 核心隐喻：人生如爬山

页面以一条**“爬山曲线”**作为主视觉，从我小学几岁开始一路向今天延伸：
- **高点（peak）**：accomplishment 多、人生高光的阶段
- **低点（valley）**：低谷期、转折期、迷茫期
- 曲线的上下起伏 = 人生的真实轨迹

### 节点（events）的两种形态

曲线上分布着我人生中的一个个事件，分两类：

1. **简短节点（majority）**
   - 一到两句话就能讲完的经历（没有当时的细节或照片）
   - 通过一条**引出线**从曲线上的点拉出，连到一个小气泡 / 方框，里面写一句话
   - 不可点击，仅作为时间线上的一个标记

2. **可展开节点（few）**
   - 有故事、有照片、值得深入讲的经历
   - 点击后进入详细页面或展开层（具体形式待定）
   - 视觉上要和简短节点有区分（更突出 / 有 hover 状态 / 图标提示等）

### 设计原则

- 必须**有趣 + 有创意**，不要做成普通的垂直 timeline
- 必须解决**信息密度问题**：当节点很多时，引出来的方框不能让页面显得凌乱
- 必须考虑**手机竖屏体验**：曲线和节点布局都要适配窄屏
- 美学风格 **TBD**（待与用户讨论后确定）

👉 目标：**让网站有记忆点，而不是一个普通简历站**

# 十四、已知 CSS / 浏览器陷阱（避免重复踩坑）

## 1️⃣ `transform` 与 `mask-image` 不能放在同一个元素
WebKit/Chrome 会在合成时丢失 mask。

✅ 正确做法：mask 放在外层容器 `<div>`，transform 放在内部元素（例如 `<video>`）。

## 2️⃣ `position: fixed` 会被祖先的 `transform` 困住
当任意祖先元素有 `transform`、`filter`、`will-change` 时，子元素的 `position: fixed` 会变成相对该祖先定位，而不是相对视口。

✅ 正确做法：modal / lightbox / 全屏遮罩必须用 `ReactDOM.createPortal` 渲染到 `document.body`。

## 3️⃣ 单个 video / image 的视觉微调
**不要**为单个媒体元素写专门的 JSX 分支。在 `src/app/tldr/[slug]/page.tsx` 的 `LOCAL_MEDIA` 中已有的 per-item 字段模式（`transform` / `scaleX` / `maskFadeX` 等）上扩展即可。
